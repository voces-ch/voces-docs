# Create a campaign

Once your organization is registered, you can create your first campaign (for example a petition or open letter).

## How to start

1. In the left main menu, click **Campaigns**.
2. In the top-right corner, click **Create**.

You are now in the campaign editor. The form is split into three clear sections.

## 1. Basic information

Define your campaign's core settings here.

- **Is Active:** Pause or activate a campaign anytime.
- **Title:** Public campaign name (for example _Save Silver Lake_).
- **Submit Label:** Text shown on the submit button. Default is "Sign now", but you can customize it.
- **Signature Goal:** Number of signatures you want to collect. If set (for example `1000`), you can display a motivating progress bar in the widget.

## Data pooling (advanced feature)

::: danger Warning: for experienced users only
This feature changes data visibility and legal responsibility in your campaign architecture. Enable pooling only if you fully understand the privacy implications and have aligned with all partner organizations.
:::

By default, voces uses strict data separation: you (the host) only see data collected through your own host form. A partner only sees data collected through their own partner source.

With **Data Pooling**, host-side visibility expands: as host, you can read **all** collected signatures, including those generated by partners. Partners still only see their own data.

### Legal responsibility

If you enable this, it is your responsibility to ensure signers on partner forms are clearly informed that their data is also shared with the host. Privacy notices and consent language must be adapted accordingly.

### How to enable pooling

To avoid accidental privacy violations, pooling is locked by default.

1. Click **Enable Pooling** (open lock icon) above the switch.
2. Read and confirm the legal warning in the modal.
3. Pooling is then automatically turned on.

To stop pooled sharing, click **Disable Pooling** (closed red lock icon). Data silos are restored immediately.

## 2. Success action

Define what should happen after someone clicks submit:

- **Show a success message:** The form is replaced by a text message. A default is provided, but fully customizable.
- **Redirect to URL:** Send signers to a dedicated thank-you page or donation page. Enter full URL including `https://`.
